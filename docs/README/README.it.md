# WhisperColab

Quaderni multilingue Google Colaber la trascrizione Whisper.

## Caratteristiche

- **Gratuito da usare:** Questo notebook utilizza [openai/whisper](https://github.com/openai/whisper), reso open source da OpenAI il 17 gennaio 2023, insieme al runtime GPU gratuito di Google Colab.
- **Nessun limite fisso per le dimensioni dei file sul lato notebook:** È possibile elaborare file audio e video di grandi dimensioni.
- **Elaborazione batch:** È possibile elaborare più file audio e video in un'unica esecuzione.
- **Supporto per Google Drive:** I file possono essere caricati direttamente oppure i file audio/video archiviati in Google Drive possono essere elaborati.
- **Completamente online:** I download e l'elaborazione vengono eseguiti su Google Colab. Non vengono utilizzate risorse informatiche locali.
- **Formati di output multipli:** Il notebook genera la trascrizione JSON originale e i file di sottotitoli TXT, CSV, Markdown, Excel e SRT derivati.

## Disposizione del deposito

```text
.github/
  workflows/
    ci.yml
docs/
  README/
    README.<language-code>.md
locales/
  *.json
notebook_template/
  Whisper_Colab.template.ipynb
notebooks/
  Whisper_Colab_<language-code>.ipynb
scripts/
  build_notebooks.py
  build_readmes.py
  check_notebooks.py
  languages.py
  repair_translated_locales.py
  sync_locales.py
  translate_full_locales.py
.gitattributes
.gitignore
LICENSE
README.md
```

The root READMEis the English entry point, and docs/READMEcontains generated localized READMEfiles. The template notebook is the source notebook and contains placeholders such as `{{cell1.title}}`. Locale files provide language-specific READMEand Colab-facing copy. I taccuini generati vengono impegnati in modo che i collegamenti Colab rimangano stabili.

## Costruisci

```bash
python scripts/sync_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## Controlli

```bash
python scripts/check_notebooks.py
```

Il flusso di lavoro CI ricostruisce file READMEe notebook dal modello e dai file locali, controlla il notebook JSON, verifica gli output vuoti, controlla la sintassi Python nelle celle di codice, esegue la scansione per evidenti segreti accidentali e fallisce quando i file generati non vengono sottoposti a commit.

## Cosa può testare l'IC

- READMEgenerazione dai file locali
- notebook generation from the template and locale files
- taccuino JSONvalidità
- output di esecuzione vuoti
- Sintassi Python nelle celle di codice
- evidenti segreti accidentali
- committed generated READMEand notebook files matching the sources

Il percorso di trascrizione completo richiede API specifiche di Colab, download di modelli, disponibilità GPUe file multimediali forniti dall'utente. Treat that path as a manual Colab smoke test after generated-file checks pass.
