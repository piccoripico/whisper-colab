# WhisperColab

Cahiers Google Colabmultilingues pour la transcription Whisper.

## Caractéristiques

- **Utilisation gratuite:** Ce bloc-notes utilise [openai/whisper](https://github.com/openai/whisper), lancé en open source par OpenAI le 17 janvier 2023, ainsi que le moteur d'exécution GPU gratuit de Google Colab.
- **Aucune limite fixe de taille de fichier côté ordinateur portable:** Des fichiers audio et vidéo volumineux peuvent être traités.
- **Traitement par lots:** Plusieurs fichiers audio et vidéo peuvent être traités en une seule fois.
- **Assistance Google Drive:** Les fichiers peuvent être téléchargés directement ou les fichiers audio/vidéo stockés dans Google Drive peuvent être traités.
- **Entièrement en ligne:** Les téléchargements et le traitement sont effectués sur Google Colab. Aucune ressource informatique locale n'est utilisée.
- **Plusieurs formats de sortie:** Le bloc-notes génère la transcription JSON originale et les fichiers de sous-titres dérivés TXT, CSV, Markdown, Excel et SRT.

## Disposition du référentiel

```text
.github/
  workflows/
    ci.yml
docs/
  README/
    README.<language-code>.md
locales/
  *.json
notebook_template/
  Whisper_Colab.template.ipynb
notebooks/
  Whisper_Colab_<language-code>.ipynb
scripts/
  build_notebooks.py
  build_readmes.py
  check_notebooks.py
  languages.py
  repair_translated_locales.py
  sync_locales.py
  translate_full_locales.py
.gitattributes
.gitignore
LICENSE
README.md
```

La racine READMEest le point d'entrée en anglais et docs/READMEcontient les fichiers READMElocalisés générés. Le bloc-notes modèle est le bloc-notes source et contient des espaces réservés tels que `{{cell1.title}}`. Les fichiers de paramètres régionaux fournissent une copie READMEspécifique à la langue et orientée Colab. Les notebooks générés sont validés afin que les liens Colab restent stables.

## Construire

```bash
python scripts/sync_locales.py
python scripts/build_readmes.py
python scripts/build_notebooks.py
```

## Chèques

```bash
python scripts/check_notebooks.py
```

Le flux de travail CI reconstruit les fichiers READMEet les blocs-notes à partir du modèle et des fichiers de paramètres régionaux, vérifie le bloc-notes JSON, vérifie les sorties vides, vérifie la syntaxe Python dans les cellules de code, recherche les secrets accidentels évidents et échoue lorsque les fichiers générés ne sont pas validés.

## Ce que CI peut tester

- READMEgénération à partir de fichiers de paramètres régionaux
- génération de notebook à partir des fichiers de modèle et de paramètres régionaux
- carnet JSONvalidité
- sorties d'exécution vides
- Syntaxe Python dans les cellules de code
- secrets accidentels évidents
- READMEgénéré par commit et fichiers notebook correspondant aux sources

Le chemin de transcription complet nécessite des API spécifiques à Colab, des téléchargements de modèles, une disponibilité GPUet des fichiers multimédias fournis par l'utilisateur. Traitez ce chemin comme un test de fumée Colab manuel une fois les vérifications des fichiers générés réussies.
